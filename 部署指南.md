# TikTok 连接器部署指南

## 版本信息

- **版本**: v2.0
- **描述**: 同步 TikTok 广告、视频和用户数据，支持多账户管理和实时 Webhook 事件订阅
- **分类**: Marketing

## 环境要求

### 1. 基础环境

- **Java**: JDK 17
- **Maven**: 3.8+
- **PostgreSQL**: 15+
- **Nacos**: 2.5.1+
- **Docker**: 20.10+ (可选)
- **Docker Compose**: 2.0+ (可选)

### 2. 硬件要求

- **CPU**: 2 核心以上
- **内存**: 4GB 以上
- **磁盘**: 20GB 以上

## 部署步骤

### 1. 安装 PostgreSQL 数据库

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install postgresql-15

# CentOS/RHEL
sudo yum install postgresql15-server
sudo postgresql-setup initdb
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

### 2. 创建数据库和用户

```bash
# 切换到 postgres 用户
sudo -u postgres psql

# 创建数据库
CREATE DATABASE livepulse_tiktok;

# 创建用户
CREATE USER tiktok_user WITH PASSWORD 'your_password';

# 授权
GRANT ALL PRIVILEGES ON DATABASE livepulse_tiktok TO tiktok_user;

# 退出
\q
```

### 3. 执行数据库脚本

```bash
# 进入 SQL 脚本目录
cd livepulse-connector-tiktok-open/docs/sql

# 执行建表脚本
psql -U tiktok_user -d livepulse_tiktok -f tiktok_schema.sql
```

### 4. 配置 Nacos

#### 4.1 导入配置

在 Nacos 控制台创建以下配置：

**Data ID**: `postgresql-config.yml`
**Group**: `DEFAULT_GROUP`

```yaml
spring:
  datasource:
    driver-class-name: org.postgresql.Driver
    url: jdbc:postgresql://postgresql-host:5432/livepulse_tiktok
    username: tiktok_user
    password: your_password
    hikari:
      minimum-idle: 5
      maximum-pool-size: 20
      idle-timeout: 300000
      connection-timeout: 30000
      max-lifetime: 1200000

mybatis-plus:
  mapper-locations: classpath*:/mapper/**/*.xml
  type-aliases-package: com.cs.entity
  configuration:
    map-underscore-to-camel-case: true
    cache-enabled: true
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
  global-config:
    db-config:
      id-type: auto
      logic-delete-field: deleted
      logic-delete-value: 1
      logic-not-delete-value: 0
```

**Data ID**: `kafka-config.yml`
**Group**: `DEFAULT_GROUP`

```yaml
spring:
  kafka:
    bootstrap-servers: kafka-host:9092
    consumer:
      group-id: tiktok-connector-group
      auto-offset-reset: latest
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

### 5. 构建项目

```bash
# 进入项目根目录
cd livepluse-connectorx

# 编译打包
mvn clean package -DskipTests

# 查看生成的 JAR 文件
ls -lh livepulse-connector-tiktok-open/target/livepulse-connector-tiktok-open.jar
```

### 6. 配置应用

创建 `application-prod.yml` 文件：

```yaml
server:
  port: 23012

spring:
  application:
    name: livepulse-connector-tiktok-open

# TikTok 定时任务配置
tiktok:
  scheduled:
    enabled: true
    ad-sync:
      cron: "0 0 */4 * * ?"  # 每4小时执行一次
    video-sync:
      cron: "0 0 2 * * ?"  # 每天凌晨2点
    user-sync:
      cron: "0 0 3 * * ?"  # 每天凌晨3点
    insight-sync:
      cron: "0 0 */6 * * ?"  # 每6小时执行一次
```

### 7. 启动应用

#### 方式一：直接启动

```bash
java -jar livepulse-connector-tiktok-open/target/livepulse-connector-tiktok-open.jar \
  --spring.profiles.active=prod \
  --NACOS_SERVER_ADDR=nacos-host:8848
```

#### 方式二：使用 Docker 启动

```bash
# 构建镜像
cd livepulse-connector-tiktok-open
docker build -t livepulse-connector-tiktok:2.0 .

# 启动容器
docker-compose up -d

# 查看日志
docker-compose logs -f
```

### 8. 配置 Nginx（可选）

```nginx
upstream tiktok-connector {
    server localhost:23012;
}

server {
    listen 80;
    server_name tiktok.yourdomain.com;

    location / {
        proxy_pass http://tiktok-connector;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## TikTok API 配置

### 1. 创建 TikTok 应用

1. 访问 [TikTok for Developers](https://developers.tiktok.com/)
2. 创建新的应用
3. 获取 API Key 和 API Secret
4. 配置 OAuth 重定向 URL

### 2. 获取访问令牌

1. 使用 OAuth 2.0 流程获取授权码
2. 使用授权码换取访问令牌
3. 定期刷新访问令牌

### 3. 配置 TikTok 账户连接

通过 API 添加 TikTok 账户配置：

```bash
curl -X POST http://localhost:23012/tiktok/account \
  -H "Content-Type: application/json" \
  -d '{
    "accountId": "tiktok_123456789",
    "accountName": "My Account",
    "displayName": "My TikTok Account",
    "accessToken": "your-access-token-here",
    "appId": "your_app_id",
    "appSecret": "your_app_secret",
    "advertiserId": "your_advertiser_id",
    "enabled": true
  }'
```

## 故障排查

### 1. 检查日志

```bash
# Docker 日志
docker-compose logs -f

tail -f logs/livepulse-connector-tiktok-open.log
```

### 2. 常见问题

#### 问题 1：无法连接到 PostgreSQL

检查数据库连接配置：

```bash
psql -U tiktok_user -d livepulse_tiktok -h postgresql-host
```

#### 问题 2：无法注册到 Nacos

检查 Nacos 地址配置：

```bash
curl http://nacos-host:8848/nacos/v1/ns/instance/list?serviceName=tiktok-open-connector-server
```

#### 问题 3：TikTok API 调用失败

检查 API 凭据配置：
- 确认访问令牌有效
- 确认应用有足够的权限
- 确认账户 ID 格式正确

## 数据模型说明

### 核心表结构

1. **tiktok_account**: TikTok 账户配置
2. **tiktok_ad**: 广告数据
3. **tiktok_video**: 视频数据
4. **tiktok_user**: 用户数据
5. **tiktok_insight**: 洞察数据（展示、点击、花费等）
6. **tiktok_webhook_log**: Webhook 日志
7. **tiktok_sync_log**: 同步日志

### 同步频率

- 广告：每4小时同步最近30天数据
- 视频：每天凌晨2点同步最近30天数据
- 用户：每天凌晨3点同步最近30天数据
- 洞察数据：每6小时同步最近7天数据

## 安全建议

1. **敏感数据加密**：对访问令牌进行加密存储
2. **访问控制**：配置 Nginx IP 白名单
3. **HTTPS 配置**：生产环境使用 SSL 证书
4. **日志脱敏**：避免在日志中输出敏感信息
5. **令牌轮换**：定期更新访问令牌
